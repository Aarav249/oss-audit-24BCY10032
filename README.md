## Scripts — What Each One Does

### Script 1 — System Identity Report (`script1_system_identity.py`)
Displays a welcome screen showing the Linux distro, kernel version, current user, home directory, date/time, and the open-source license of the OS. Think of it as the system introducing itself.

**Concepts used:** variables, command substitution equivalents, formatted output, platform/getpass/datetime/os modules.

### Script 2 — FOSS Package Inspector (`script2_package_inspector.py`)
Checks whether Python 3 is installed on the system, finds its version and binary location, retrieves package manager details via `dpkg`, and uses a dictionary-based case statement to print a philosophy note about the package.

**Concepts used:** if-else conditional, case statement (dictionary), subprocess for shell commands, pipe with grep equivalent.

### Script 3 — Disk and Permission Auditor (`script3_disk_auditor.py`)
Loops through the filesystem and reports disk usage (via `df -h`), current directory size (via `du -sh`), and file permissions (via `ls -la`). Also checks and prints the permissions of Python's config directory.

**Concepts used:** for loop over a list, os.path.exists() for directory checking, subprocess for system commands.

### Script 4 — Log File Analyzer (`script4_log_analyzer.py`)
Reads `/var/log/syslog` line by line, asks the user to enter a keyword, counts how many lines match, and prints the last 5 matching lines. Requires sudo to access the syslog file.

**Concepts used:** while-read loop equivalent, if-then keyword matching, counter variables, command-line arguments, user input.

### Script 5 — Open Source Manifesto Generator (`script5_manifesto_generator.py`)
Asks the user three questions interactively, composes a personalised open-source philosophy statement from their answers, saves it to a `.txt` file named after the current user, and prints it back.

**Concepts used:** read equivalent (input()), string concatenation, file writing (> equivalent), date command, reading file back (cat equivalent).

## How to Run the Scripts

### Requirements
- Python 3.x (tested on Python 3.12.3)
- Ubuntu / Debian Linux (or WSL2 on Windows)
- No additional packages needed — only Python standard library is used

### Steps

**1. Clone the repository**
```bash
git clone https://github.com/[your-username]/oss-audit-[rollnumber].git
cd oss-audit-[rollnumber]/scripts
```

**2. Run Script 1 — System Identity**
```bash
python3 script1_system_identity.py
```

**3. Run Script 2 — Package Inspector**
```bash
python3 script2_package_inspector.py
```

**4. Run Script 3 — Disk Auditor**
```bash
python3 script3_disk_auditor.py
```

**5. Run Script 4 — Log Analyzer** *(requires sudo)*
```bash
sudo python3 script4_log_analyzer.py
# When prompted, type: error  (or any keyword like warning, fail)
```

**6. Run Script 5 — Manifesto Generator**
```bash
python3 script5_manifesto_generator.py
# Answer the three questions when prompted
# Your manifesto will be saved as manifesto_[yourusername].txt
```

## Notes

- Scripts were developed and tested on **Ubuntu 24 (WSL2)** running on Windows 11.
- Script 4 reads `/var/log/syslog` which may require `sudo` on some systems. On systems using `journald` instead of syslog, you may need to change the log path to `/var/log/kern.log` or use `journalctl` output.
- The manifesto file generated by Script 5 will be named after your current Linux username (e.g., `manifesto_aarav.txt`).
- Script files are saved with `.py` extension but contain the logic described in the `.sh` starter templates from the project brief.
